When you use external stylesheets, all that's needed is the CSS. In other words, take out the <style type="text/css"> and </style>. Those are only required for embedded stylesheets :)

PS. When using a url() value in your CSS, it's a good idea to put the filename in quotes, because some browsers won't read it without them.

Like this:     background-image: url(The%20Internet%20Classroom%20-%20Not%20Found-31.html);
Not like this: background-image: url(The%20Internet%20Classroom%20-%20Not%20Found-30.html);

Well, I don't know what the problem is exactly, but here are a couple things I see:

  • Leave a space before the closing slash in the <meta> element.

  • Separate your font choices with commas, not semicolons.

  • Put quotation marks around font names with more than one word, eg. "Lucida Calligraphy" (and make sure the comma comes after the closing quotation mark, even though it goes against the rules of English grammar :P)

2006/06/25 00:28:06 PDT by aqhong

I Googled it too, and according to the following pages, Mac IE 5 reads double quotes, but not single quotes.

http://www.communis.co.uk/dithered/css_filters/css_only/import_double_quotes.html
http://www.communis.co.uk/dithered/css_filters/css_only/import_single_quotes.html

It appears that the only browsers that have problems with any kind of quotes are IE4 and NS4 on both platforms. And the CSS support on those browsers is so bad that feeding them advanced CSS (that is, anything other than basic font and color styling) would probably cause them to mangle your page beyond recognition, which is even worse than having a page with no style. So I would still recommend using double quotes over no quotes at all, unless getting your CSS to display in a decade-old browser, however badly, is a priority for you :P

Server crashed a some posts were lost. Lemme recap:

Links (the "a" tag) are inline elements. The width of inline elements is exactly equal to the width of what's inside (text, in this case). So the linked text is actually centered. If you want to center the link with respect to the page, you'll have to apply "text-align: center" to a block element like paragraphs or lists.
